--- title: Internationalization features at Facebook layout: posts categories: projects permalink: /projects/facebook --- During my internship at Facebook, I worked on three main features for the Internationalization (i18n) team:

  • Integrating Android data into the existing i18n data pipeline
  • Implementing a "See Conversion" feature for user-generated text that should be converted to a readable format for a user
  • Creating A/B tests for UI changes to the language switchers on the Android client

The data work consisted of querying and modifying existing data pipelines with Apache Hive and SQL, to process and account for a set of Android data that was previously collected, but not analyzed. This allowed for certain i18n services to be much more accurate, since many Facebook users use the Android client.

"See Conversion" was a feature for Facebook's mobile site that was necessary due to the incompatibility of certain languages' representations with Unicode encodings. For example, Zawgyi is the most widely used encoding for input in Myanmar-language sites, but is not Unicode-compliant, resulting in incorrect renderings on Unicode devices. This conversion tool allows Zawgyi to be converted to Unicode in posts, similar to the "See Translation" tool.
